# Описание проекта «Мотт Ларбе» для дизайна

Документ предназначен для создания дизайна продукта в отдельном проекте (Claude и др.). Содержит точное описание текущего состояния backend, целевой функциональности и карты страниц.

---

## 1. Суть продукта

**Название:** Мотт Ларбе (языковая платформа).

**Цель:** Платформа для изучения языка через чтение реальных текстов с мгновенным переводом по клику, морфологическим разбором и персональным словарём.

**Ключевая идея:** Чтение текста = интерактивное изучение. Каждое слово — точка входа в словарь: клик → перевод, базовая форма, грамматическая форма. Без переключения вкладок и поиска в словаре.

**Решаемые проблемы:**

- Постоянное переключение между текстом и словарём
- Потеря контекста при переводе
- Сложность морфологии (особенно редкие/малоизвестные языки)
- Неудобство работы с адаптированными текстами

## 3. Архитектура backend (модули)

Backend построен как набор движков:

| Движок / модуль          | Назначение                                                                                          |
| ------------------------ | --------------------------------------------------------------------------------------------------- |
| **Text**                 | Тексты, страницы, версии, хранение Tiptap JSON и plain-текста                                       |
| **Tokenizer**            | Разбиение текста на токены (слова), привязка к страницам                                            |
| **Normalizer**           | Приведение слов к нормализованной форме (lowercase, без пунктуации)                                 |
| **Dictionary (Admin)**   | Админский словарь: Lemma, Headword, MorphForm, Sense, DictionaryEntry                               |
| **Dictionary Cache**     | Кэш переводов по нормализованной форме для быстрого поиска                                          |
| **Online Dictionary**    | Интеграция внешнего словаря (импорт/поиск)                                                          |
| **Morphology**           | Морфологический анализ, базовая форма, правила (Rule Engine, Importer, Cleaner)                     |
| **Unknown Word**         | Логирование слов, которых нет в словаре (для расширения словаря)                                    |
| **Token**                | Информация по токену: перевод, грамматика, базовая форма (в т.ч. кэш)                               |
| **Words**                | Lookup по `tokenId` (клик по слову) и lookup по строке (`lookup-by-word`)                           |
| **Progress**             | Прогресс по тексту и по словам (UserTextProgress, UserWordProgress)                                 |
| **Deck**                 | Авторская система заучивания: деки NEW/OLD/RETIRED/NUMBERED, авторебалансировка, ежедневная ротация |
| **Dictionary (User)**    | Личный словарь пользователя: слова, папки, уровни изучения, статистика                              |
| **Auth**                 | Login, Register, Refresh, Logout, JWT, роли USER/ADMIN                                              |
| **User**                 | Профиль, обновление, удаление аккаунта                                                              |
| **Admin: Billing**       | Планы, подписки, платежи, купоны, возвраты                                                          |
| **Admin: Users**         | Управление пользователями, RBAC-роли, статусы, события                                              |
| **Admin: Analytics**     | Аналитика: сложные тексты, популярные уровни                                                        |
| **Admin: Feature Flags** | Управление фичафлагами (глобально и per-user)                                                       |
| **Phrasebook**           | Разговорник: категории и фразы с разбором по словам, примерами, сохранениями и предложениями        |

Обработка текста после сохранения: извлечение plain-текста из Tiptap → токенизация → нормализация → поиск в словаре/морфологии/кэше → связь токенов со словарём (TextVocabulary, TokenAnalysis).

## 4. Модель данных (основные сущности)

### Пользователи и роли

- **User:** id, email, password, username, name, surname, phone?, role (USER | ADMIN), status (ACTIVE | BLOCKED | FROZEN | DELETED), refresh token
- **UserRole:** USER, ADMIN (+ RBAC-роли через таблицу ролей и разрешений)
- **PermissionCode:** CAN_MANAGE_BILLING, CAN_VIEW_ANALYTICS, CAN_MANAGE_USERS, CAN_MANAGE_FEATURE_FLAGS, CAN_MANAGE_TEXTS, CAN_MANAGE_DICTIONARY, CAN_MANAGE_MORPHOLOGY

### Тексты и теги

- **Text:** id, title, level?, language (CHE | RU), author, source?, imageUrl?, createdById
- **Tag:** id, name (уникальное) — глобальный справочник тегов (Литература, История, Сказки и т.д.)
- **TextTag:** textId + tagId — связь many-to-many между Text и Tag
- **TextPage:** id, textId, pageNumber, title? (название главы/раздела), contentRich (Tiptap JSON), contentRaw (plain)
- **TextProcessingVersion:** версия обработки текста (textId, version)

### Токены и словарь текста

- **TextToken:** id, versionId, pageId?, position, original, normalized, status (ANALYZED | AMBIGUOUS | NOT_FOUND), vocabId? → связь с TextVocabulary
- **TextVocabulary:** на уровне версии текста: versionId, normalized, lemmaId?, translation? (связь токенов с леммами и переводами)

### Словарь (админ + импорт)

- **Lemma:** базовая форма слова (baseForm, normalized, language, partOfSpeech?, frequency?)
- **DictionaryEntry:** запись из внешнего/админского словаря (rawWord, rawTranslate, notes, source: ADMIN | IMPORT | ONLINE | CACHE)
- **Headword:** форма входа в словарь (entryId, text, normalized, lemmaId?)
- **MorphForm:** словоформа (form, normalized, grammarTag?, lemmaId, entryId?)
- **Sense:** значение/определение (entryId, order, definition, notes)
- **Example:** пример к значению (senseId, text, translation?)

### Кэш и анализ

- **DictionaryCache:** быстрый поиск по normalized (normalized, lemmaId?, translation?)
- **TokenAnalysis:** какой леммой разобран токен (tokenId, lemmaId?, source: ADMIN | CACHE | ONLINE | MORPHOLOGY, probability?, isPrimary?)

### Неизвестные слова

- **UnknownWord:** word, normalized, seenCount, firstSeen, lastSeen (уникальность по normalized)

### Личный словарь пользователя (реализовано)

- **UserDictionaryEntry:** id, userId, word, translation, normalized, lemmaId?, folderId?, learningLevel (NEW | LEARNING | KNOWN), repetitionCount, addedAt
- **UserDictionaryFolder:** id, userId, name — папки для организации слов
- Статистика: total, byLevel {NEW/LEARNING/KNOWN}, totalRepetitions

### Прогресс пользователя

- **UserTextProgress:** userId, textId, progressPercent, lastOpened
- **UserWordProgress:** userId, lemmaId, status (NEW | LEARNING | KNOWN), seenCount, repetitions, lastSeen, nextReview, easeFactor?, interval? (для интервального повторения)

### Деки (авторская система заучивания)

- **UserDeckCard:** id, userId, lemmaId, deckType (NEW | OLD | RETIRED | NUMBERED), deckNumber? (только для NUMBERED), movedAt
- **UserDeckState:** userId, currentNumberedDeck, lastRotatedAt — состояние ежедневной ротации нумерованных дек

### Разговорник (реализовано)

- **PhrasebookCategory:** id, emoji, name, sortOrder
- **PhrasebookPhrase:** id, categoryId, original, transliteration?, translation, lang (CHE | RU), sortOrder
- **PhrasebookPhraseWord:** id, phraseId, original, translation, position — разбор фразы по словам
- **PhrasebookPhraseExample:** id, phraseId, phrase, translation, context? — примеры использования
- **UserPhrasebookSave:** userId + phraseId — сохранённые фразы пользователя
- **PhrasebookSuggestion:** userId?, original, translation, lang, context?, categoryId? — предложения фраз от пользователей

### Аналитика и события (реализовано)

- **UserEvent:** userId, type (ADD_TO_DICTIONARY | FAIL_LOOKUP | OPEN_TEXT | …), metadata (JSON), createdAt
- Агрегации: количество событий по типу, топ неизвестных слов (topFailLookups), топ кликов

### Биллинг (реализовано)

- **Plan:** id, name, price, interval (MONTHLY | YEARLY | LIFETIME), isActive, features
- **Subscription:** id, userId, planId, status (ACTIVE | CANCELED | EXPIRED | TRIAL), startDate, endDate?, trialDays?, isLifetime
- **Payment:** id, userId, subscriptionId?, planId?, amount, status (PAID | REFUNDED | PARTIAL_REFUND), paidAt, refundedAt?
- **Coupon:** id, code, discountPercent?, discountFixed?, maxUses?, redeemedCount, expiresAt?, isActive
- **CouponRedemption:** couponId, userId, paymentId?

### Feature Flags (реализовано)

- **FeatureFlag:** id, name, description?, isEnabled (глобально)
- **UserFeatureFlagOverride:** flagId, userId, isEnabled (переопределение per-user)

**GET /api/texts — query-параметры:**

| Параметр | Значения | Описание |
| -------- | -------- | -------- |
| `language` | `CHE`, `RU`, `AR`, `EN` | Один или несколько (повторить параметр). Без параметра — все языки. |
| `level` | `A1`–`C2` | Один или несколько (повторить параметр). Без параметра — все уровни. |
| `tagId` | UUID | Один или несколько (повторить параметр). Тексты с хотя бы одним из указанных тегов. |
| `status` | `NEW`, `IN_PROGRESS`, `COMPLETED` | Фильтр по статусу прогресса (только с авторизацией). |
| `orderBy` | `newest`, `oldest`, `alpha`, `progress`, `length`, `level` | Сортировка. По умолчанию: `newest`. `level` — A1→C2. |
| `search` | строка | Поиск по названию и автору (case-insensitive). |

**Статусы прогресса:** `NEW` = не начат (0%), `IN_PROGRESS` = читается (1–99%), `COMPLETED` = завершён (100%).

**Ответ GET /api/texts:**
```json
{
  "items": [{ "id", "title", "level", "language", "author", "imageUrl", "tags": [{ "id", "name" }],
              "wordCount", "readingTime", "progressPercent", "progressStatus", "lastOpened", "isNew" }],
  "counts": { "total", "new", "inProgress", "completed" }
}
```

> `isNew: true` если текст опубликован в последние 30 дней. Возвращается всегда, авторизация не нужна.
```

**GET /api/texts/continue-reading — ответ включает:** `currentPage`, `totalPages`, `progressPercent`, `lastOpened`, `imageUrl`, `tags`, `readingTime`, `wordCount`.

**GET /api/texts/:id — ответ включает (все пользователи):** `wordCount`, `readingTime`, `totalPages`, `pages[]` (`id`, `pageNumber`, `title`), `publishedAt`, `imageUrl`, `source`, `tags`.
**GET /api/texts/:id — только при авторизации:** `progressPercent`, `lastOpened`, `currentPage`, `wordStats` (`total`, `known`, `learning`, `new`), `isFavorite`.

**GET /api/texts (ответ) — поле `isFavorite`:** возвращается в каждом элементе списка. `true` если пользователь добавил текст в закладки. Без токена: `false`.

**GET /api/texts/:id/pages/:pageNumber — ответ включает:** `totalPages`, `wordCount`, `page` (`id`, `pageNumber`, `title`, `contentRich`, `contentRaw`), `tokens[]` с полями `lemmaId` и `userStatus`, `progress`.

**POST /api/words/lookup — ответ включает:** `lemmaId`, `translation`, `tranAlt`, `grammar`, `baseForm`, `forms[]`, `tags[]`, `examples[]`, `userStatus`, `inDictionary`, `dictionaryEntryId`.
- `inDictionary: true` — слово уже добавлено в личный словарь; кнопка показывает «В словаре»
- `dictionaryEntryId` — ID записи в личном словаре (нужен для `DELETE /api/dictionary/:id`)
- `tranAlt` — дополнительный перевод/уточнение (например, второе значение)
- `tags[]` — теги: часть речи + грамматический тег формы (падеж, число и т.д.)

## 7. Ключевые сценарии для UI/UX

### 7.1 Чтение текста и клик по слову

1. Пользователь открывает текст (Library → Text details → Reader).
2. Видит страницу с интерактивными словами (контент + токены с `GET /api/texts/:id/pages/:pageNumber`).
   - Каждый токен содержит `lemmaId` и `userStatus` (NEW/LEARNING/KNOWN/null) — для подсветки цветом.
3. Кликает по слову → frontend вызывает **POST `/api/words/lookup`** с `{ "tokenId": "..." }`.
4. Backend: поиск по tokenId → цепочка Admin Dict → Cache → Online → Morphology; при отсутствии — запись в UnknownWord (seenCount+1); событие FAIL_LOOKUP в UserEvent.
5. Ответ: `translation`, `tranAlt`, `grammar`, `baseForm`, `forms[]`, `tags[]`, `examples[]`, `userStatus`, `inDictionary`, `dictionaryEntryId` — показываются в Word popup или в боковой панели (Word panel).
6. Кнопка «Добавить в словарь» / «В словаре»:
   - `inDictionary: false` → кнопка «Добавить» → **POST `/api/dictionary`** с `{ tokenId }` → learningLevel=NEW; событие ADD_TO_DICTIONARY.
   - `inDictionary: true` → кнопка «В словаре» → клик → **DELETE `/api/dictionary/:dictionaryEntryId`**.
7. Закладка текста: кнопка «В избранное» → **POST `/api/texts/:id/bookmark`** (toggle). Ответ: `{ bookmarked: boolean }`.

### 7.2 Личный словарь

- Список слов: слово, перевод, уровень (NEW / LEARNING / KNOWN), дата добавления, папка.
- Фильтры по уровню и папке; поиск по слову.
- Карточка слова: изменить уровень, переместить в папку, удалить.
- Папки: создание, переименование, удаление; слова привязаны к папкам опционально.
- Статистика вверху: итого / по уровням / суммарные повторения.

### 7.3 Неизвестное слово

- Если слово не найдено: в ответе пустые/нулевые поля; на backend создаётся/обновляется UnknownWord.
- В админке раздел **Unknown words**: список (слово, нормализованная форма, количество появлений, контекст/текст, даты). Действия: добавить в словарь, связать с существующей леммой, удалить.

### 7.4 Админ: тексты

- Список текстов: название, уровень, язык, количество слов, дата, статус публикации.
- Создание/редактирование: title, language, level, author, source, страницы (Tiptap JSON).
- Отдельно: управление версиями текста, при необходимости — переразметка (re-tokenize).

### 7.5 Админ: словарь и морфология

- Словарь: поиск, карточка слова, добавление/редактирование перевода, грамматики, словоформ.
- Морфологические правила: список, добавление/редактирование, тест анализа.

### 7.6 Админ: пользователи

- Список: таблица с поиском, фильтрами (статус, роль), пагинацией.
- Карточка пользователя: профиль, RBAC-роли, статус (ACTIVE/BLOCKED/FROZEN/DELETED), подписки, история событий.
- Действия: заблокировать, заморозить, разблокировать, удалить, сбросить сессии, назначить/отозвать роли.
- События пользователя: лента событий (ADD_TO_DICTIONARY, FAIL_LOOKUP, OPEN_TEXT, …) с фильтрами по типу и дате.
- Сводка событий: количество кликов, кол-во неизвестных слов, топ проблемных слов.

### 7.7 Админ: биллинг

- **Планы:** список тарифов (название, цена, период, активность); создание и редактирование.
- **Подписки:** по конкретному пользователю; назначить план (с trial-периодом или lifetime), отменить, продлить.
- **Платежи:** общий список платежей; ручной возврат (полный или частичный).
- **Купоны:** список кодов (скидка в %, фиксированная скидка, лимит использований, срок); создание и обновление.

### 7.8 Админ: аналитика

- **Сложные тексты:** рейтинг текстов по числу FAIL_LOOKUP событий (много неизвестных слов); фильтр по дате.
- **Популярные уровни:** счётчик открытий текстов по уровням A1–C2; фильтр по дате.

### 7.9 Админ: feature flags

- Список флагов: название, описание, глобальный статус (вкл/выкл).
- Создание/редактирование флага.
- Переопределения per-user: включить/выключить конкретный флаг для пользователя; удалить переопределение.

### 7.10 (Планируется) Редактирование отдельного слова в тексте

- В админ-редакторе текста: клик по слову → карточка токена (original, normalized, position, vocabId). Можно изменить слово, нормализованную форму, привязку к словарю без полной переразметки текста.

## 8. Обязательные улучшения (из бэклога)

Эти пункты нужно учитывать в дизайне и приоритизации:

1. **Полная структура админ-панели:** тексты (создание, версии, публикация, уровень A1–C2), словарь, Unknown words, морфологические правила, пользователи, биллинг, аналитика, feature flags.
2. **Механизм клика по слову:** описан выше; важно отображение перевода/грамматики/базовой формы и добавление в личный словарь.
3. **Подписки:** бесплатный тариф (ограниченный перевод/личный словарь), премиум (без лимитов, статистика, повторения, сложные тексты). Варианты: Basic, Premium, Pro. _Биллинг-модуль реализован на backend._
4. **Личный словарь пользователя:** _реализован на backend._ Слово, перевод, дата добавления, уровень изучения, папки, статистика.
5. **Система повторения:** две независимые системы реализованы на backend, UI не реализован:
   - **SM-2 (интервальное повторение):** оценки 0–5, пересчёт интервала/easeFactor, автопереход в KNOWN при interval ≥ 21 дня, эффект частоты встречаемости слова в текстах.
   - **Деки (авторская):** деки NEW → OLD → RETIRED → NUMBERED (до 90 карточек каждая), автоматическое смещение при переполнении, ежедневная ротация нумерованных дек. Формат: карточки слово ↔ перевод с контекстным сниппетом из текста.
6. **Редактирование отдельных токенов в тексте:** без полной переразметки; массовое исправление при необходимости; опция «Re-tokenize text» только при сильном изменении текста.

## 9. Уже реализовано на backend

Деки (авторская система заучивания)

- Регистрация и вход (JWT + refresh в cookie), статусы пользователей (ACTIVE / BLOCKED / FROZEN / DELETED).
- CRUD текстов (с страницами в Tiptap), получение страницы с токенами. Поля `imageUrl`, `tags` у текста.
- **Теги текстов:** модели `Tag` и `TextTag`; CRUD тегов через `/api/admin/tags`; назначение тегов при создании и редактировании текста через `tagIds`.
- Список текстов (`GET /api/texts`) с мультифильтрацией по языку, уровню и тегу; фильтром по статусу прогресса (`NEW`/`IN_PROGRESS`/`COMPLETED`); сортировкой (`newest`/`oldest`/`alpha`/`progress`/`length`/`level`); поиском по названию/автору; полями `wordCount`, `readingTime`, `progressPercent`, `progressStatus`, `isNew` (опубликован ≤ 30 дней назад); счётчиками `counts { total, new, inProgress, completed }`.
- Карточка текста (`GET /api/texts/:id`): полные метаданные + список страниц с опциональными названиями (`title`), `wordCount`, `readingTime`, `totalPages`, `publishedAt`, `imageUrl`. При авторизации: `progressPercent`, `lastOpened`, `currentPage`, `wordStats { total, known, learning, new }`.
- Похожие тексты (`GET /api/texts/:id/related`): до 6 текстов того же языка с совпадением уровня или тегов.
- Поле `title` у страниц `TextPage` для названий глав/разделов (опциональное).
- Список тегов для фронта: `GET /api/texts/tags`.
- Эндпоинт «Продолжить чтение» (`GET /api/texts/continue-reading`) с `currentPage`, `totalPages`, `progressPercent`, `readingTime`, `tags`.
- Дашборд (`GET /api/dashboard/me`): агрегирует табло (тексты, словарь, стрик, рекорд стрика, разбивка недели по дням `streakDays`, слова на повторение) и список «продолжить чтение» за один запрос.
- Токенизация и нормализация при сохранении текста.
- **Клик по слову (`POST /api/words/lookup`):** возвращает `translation`, `tranAlt`, `grammar`, `baseForm`, `forms[]`, `tags[]`, `examples[]`, `userStatus`, `inDictionary`, `dictionaryEntryId`. Поддерживает необязательную авторизацию.
- **Поиск по строке (`POST /api/words/lookup-by-word`):** цепочка Admin → Cache → Online → Morphology.
- **Страница ридера (`GET /api/texts/:id/pages/:pageNumber`):** возвращает `totalPages`, `wordCount`, `page.title`, токены с `lemmaId` и `userStatus`.
- **Закладки текстов:** `GET /api/texts/bookmarks`, `POST /api/texts/:id/bookmark` (toggle). `isFavorite` в ответах `GET /api/texts` и `GET /api/texts/:id`.
- Прогресс по тексту (progressPercent).
- Прогресс по словам с алгоритмом SM-2 (UserWordProgress): оценки 0–5, интервалы, easeFactor, эффект частоты, контексты встречи слова (WordContext).
- **Деки (авторская система заучивания, полностью):** UserDeckCard, UserDeckState; деки NEW/OLD/RETIRED/NUMBERED; авторебалансировка при добавлении; ежедневная ротация нумерованных дек.
