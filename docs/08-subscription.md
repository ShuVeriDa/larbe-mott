# Подписки и биллинг

Модуль: [src/subscription/](../src/subscription/), [src/billing/](../src/billing/)

---

## Тарифные планы

| План | Описание |
|------|---------|
| `FREE` | Бесплатный — ограниченный доступ |
| `BASIC` | Базовый — расширенный доступ |
| `PRO` | Про — полный доступ |
| `PREMIUM` | Премиум — максимум возможностей |
| `LIFETIME` | Пожизненный — одноразовая оплата |

Лимиты каждого плана определены в [src/billing/plan-limits.ts](../src/billing/plan-limits.ts).
Примеры лимитов: количество текстов, слов в личном словаре, доступ к аналитике.

---

## Статусы подписки

| Статус | Описание |
|--------|---------|
| `TRIALING` | Пробный период |
| `ACTIVE` | Активная подписка |
| `CANCELED` | Отменена (ещё действует до конца периода) |
| `EXPIRED` | Закончилась |

## API Endpoints

### Публичные (для пользователей)

| Метод | URL | Описание |
|-------|-----|---------|
| GET | `/api/subscription/plans` | Список доступных тарифных планов |
| GET | `/api/subscription/my` | Моя текущая активная подписка |

## Провайдеры оплаты

Поддерживаются следующие провайдеры (хранятся в `PaymentProvider`):

| Провайдер | Описание |
|-----------|---------|
| `STRIPE` | Основная платёжная система |
| `PAYPAL` | PayPal |
| `PADDLE` | Paddle (налоговый агент) |
| `LEMONSQUEEZY` | Lemon Squeezy (для цифровых продуктов) |
| `MANUAL` | Ручное добавление (для тестов или промо) |

---

## Промокоды (Coupons)

Промокод имеет:
- Код скидки (строка)
- Размер скидки (процент или фиксированная сумма)
- Срок действия
- Максимальное количество использований

Использование промокода записывается в `CouponRedemption`.

---

## Проверка доступа к Premium функциям

Декоратор `@Premium()` в контроллерах проверяет:
1. Есть ли у пользователя активная подписка (`ACTIVE` или `TRIALING`)
2. Если нет — возвращает `403 Forbidden`

Пример защищённого маршрута:

```typescript
@Premium()
@Get('analytics')
getAnalytics() { ... }
```
