# Прогресс обучения и алгоритм SM-2

Модуль: [src/progress/](../src/progress/)

---

## Что отслеживается

1. **Прогресс по словам** — какие слова знаешь / учишь / не знаешь
2. **Прогресс по текстам** — процент прочитанности каждого текста
3. **События** — активность для аналитики (когда и что делал)

---

## Статусы слов

| Статус | Описание |
|--------|---------|
| `NEW` | Слово встретилось, но ещё не изучалось |
| `LEARNING` | В процессе изучения, требует повторений |
| `KNOWN` | Слово хорошо усвоено |

---

## Алгоритм SM-2 (интервальное повторение)

SM-2 — это научный алгоритм, который определяет, **через сколько дней** нужно повторить слово, чтобы не забыть его.

### Принцип
Если ты легко вспоминаешь слово — следующее повторение откладывается дальше.
Если забыл — нужно повторить скоро.

### Оценка (Quality)
При повторении пользователь оценивает, насколько хорошо он вспомнил слово:

| Оценка | Описание |
|--------|---------|
| 0 | Полный провал, не вспомнил |
| 1 | Неправильно, но вспомнил подсказку |
| 2 | Неправильно, но ответ был близким |
| 3 | Правильно с трудом |
| 4 | Правильно после лёгкого раздумья |
| 5 | Идеально, без раздумий |

### Параметры SM-2

| Параметр | Описание |
|---------|---------|
| `interval` | Дней до следующего повторения |
| `easeFactor` | Коэффициент лёгкости (начинается с 2.5) |
| `repetitions` | Сколько раз слово повторено подряд успешно |
| `nextReviewDate` | Дата следующего повторения |

### Как меняется интервал

```
Первое повторение (оценка ≥ 3):  interval = 1 день
Второе повторение:               interval = 6 дней
Третье и далее:                  interval = прошлый interval × easeFactor
                                  easeFactor корректируется от оценки
```

При оценке < 3 — интервал сбрасывается, нужно повторить скоро.

## API Endpoints

### Прогресс по словам
| Метод | URL | Описание |
|-------|-----|---------|
| GET | `/api/progress/review/stats` | Статистика для SM-2 intro: dueCount, learningCount, streak |
| GET | `/api/progress/review/due` | Слова, которые нужно повторить сегодня |
| POST | `/api/progress/review/:lemmaId` | Записать результат повторения (0–5) |
| GET | `/api/progress/words/:lemmaId/contexts` | Примеры слова из прочитанных текстов |

### GET /api/progress/review/stats — ответ

```json
{
  "dueCount": 12,
  "learningCount": 48,
  "streak": 7
}
```

| Поле | Описание |
|------|---------|
| `dueCount` | Слова к повторению сегодня (nextReview ≤ now, status ≠ KNOWN) |
| `learningCount` | Всего слов в статусе LEARNING |
| `streak` | Серия дней подряд (активность по UserEvent) |

### Прогресс по текстам
| Метод | URL | Описание |
|-------|-----|---------|
| GET | `/api/progress/texts` | Прогресс по всем текстам |
| POST | `/api/progress/texts/:textId` | Обновить процент прочитанности |

## Как работает повторение слов (пользовательский флоу)

```
1. Пользователь читает текст
       │
       ▼
2. Кликает на незнакомое слово → слово добавляется в прогресс со статусом NEW
       │
       ▼
3. Пользователь идёт в раздел "Повторение" (деки)
       │
       ▼
4. GET /api/progress/words/due — получает слова на сегодня
       │
       ▼
5. Для каждого слова: показываем → пользователь вспоминает → оценивает 0-5
       │
       ▼
6. POST /api/progress/words/:id/review — SM-2 пересчитывает интервал
       │
       ▼
7. Слово уходит до следующей даты повторения
```

---

## Контексты слова

`GET /api/progress/words/:lemmaId/contexts` — возвращает реальные предложения из текстов, которые читал пользователь, где встречалось это слово. Это мощный инструмент для запоминания — слово видно в контексте.
