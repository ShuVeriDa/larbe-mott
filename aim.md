# Архитектура проекта интерактивного чтения текста

---

## 1. Цель проекта

Проект предназначен для **интерактивного изучения языка через чтение текстов**.  
Пользователь читает текст (например, книгу), может кликнуть на любое слово и мгновенно получить:

- **перевод**
- **грамматическую информацию**
- **форму слова**
- **прогресс изучения**

> **Основная идея системы:** чтение текста превращается в интерактивный словарь.

Но при этом система должна быть:

- очень **быстрой**
- **экономной** по запросам
- **масштабируемой** для больших книг

Поэтому архитектура построена так, чтобы:

1. загружалась только **одна страница** текста
2. разметка текста выполнялась **заранее**
3. перевод слова запрашивался **только после клика**

Это уменьшает нагрузку на сервер и ускоряет работу интерфейса.

---

## 2. Основные принципы архитектуры

### 1️⃣ Lazy loading переводов

Перевод слова **не загружается заранее**. Он загружается только когда пользователь нажимает на слово.

**Это снижает:**

- количество API-запросов
- нагрузку на словарь
- размер ответа сервера

---

### 2️⃣ Пагинация текста

Большие тексты (книги) разбиваются на страницы. Frontend загружает: **1 страница = 1 запрос**.

**Пример:**

```
/text/123/page/1
/text/123/page/2
/text/123/page/3
```

**Это позволяет:**

- работать даже с очень большими книгами
- не загружать мегабайты текста
- быстрее рендерить страницу

---

### 3️⃣ Предварительная разметка текста

Текст размечается **один раз** при загрузке в систему. Система разбивает текст на:

- слова
- знаки препинания
- позиции

**Пример:**

| Исходный текст | Разметка |
|----------------|----------|
| *Мен йоьлла цуьнан.* | 1. Мен<br>2. йоьлла<br>3. цуьнан<br>4. . |

Это сохраняется в базе и больше не пересчитывается при каждом открытии страницы.

---

### 4️⃣ Многоуровневый поиск слова

Когда пользователь кликает слово, система ищет перевод по цепочке:

1. **Админский словарь**
2. **Кэш словаря**
3. **Онлайн словарь**
4. **Морфологический анализ**

Это уменьшает количество внешних запросов.

---

## 3. Общий поток работы системы

### Загрузка текста (один раз)

Когда администратор добавляет текст:

```
Текст
  ↓
Tiptap JSON
  ↓
Извлечение plain-текста
  ↓
Tokenization
  ↓
Сохранение токенов
  ↓
Готово
```

После этого текст уже подготовлен.

---

### Поток чтения текста пользователем

Когда пользователь открывает страницу:

```
Frontend
  ↓
GET /text/:id/page/:page
  ↓
Backend
  ↓
возвращает токены страницы
  ↓
Frontend рендерит текст
```

Никакого словаря пока не используется.

---

### Поток клика по слову

Когда пользователь нажимает на слово:

```
Frontend
  ↓
POST /word/lookup
  ↓
Backend
  ↓
  1. нормализация      ↓
  2. поиск в админском словаре  ↓
  3. поиск в кэше      ↓
  4. онлайн словарь    ↓
  5. морфология        ↓
  возврат результата
```

---

## 4. Этапы системы

### ЭТАП 1. Управление текстами

Администратор может:

- добавлять тексты
- редактировать
- удалять
- разбивать на страницы

Тексты хранятся в формате **Tiptap JSON**. Это позволяет сохранить:

- жирный текст
- заголовки
- абзацы
- форматирование

---

### ЭТАП 2. Разметка текста (Tokenization)

Текст разбивается на **токены**. Токен — это **слово** или **знак препинания**.

**Пример структуры `TextToken`:**

| Поле       | Описание   |
|------------|------------|
| `id`       | идентификатор |
| `versionId`| версия текста  |
| `position` | позиция в тексте |
| `original` | исходная форма  |
| `normalized` | нормализованная форма |
| `vocabId`  | связь со словарём |

---

### ЭТАП 3. Нормализация слов

Слово приводится к базовой форме:

- *Мен* → *мен*
- *Цуьнан* → *цуьнан*

**Удаляются:**

- заглавные буквы
- пунктуация
- лишние символы

---

### ЭТАП 4. Поиск слова в админском словаре

Система проверяет **AdminDictionary** — основной словарь проекта. Он содержит:

- переводы
- формы слов
- грамматику

---

### ЭТАП 5. Поиск в кэше словаря

Если слово не найдено — проверяется **DictionaryCache**. Кэш содержит:

- переводы из онлайн-словаря
- уже найденные слова

---

### ЭТАП 6. Онлайн словарь

Если слово отсутствует в системе — выполняется запрос к **external dictionary API**. Результат сохраняется в кэш.

---

### ЭТАП 7. Лог неизвестных слов

Если слово не найдено — оно записывается в таблицу **UnknownWord**. Это помогает:

- расширять словарь
- видеть проблемные слова

---

### ЭТАП 8. Морфологический анализ

Если слово не найдено напрямую — используется морфологический анализ. Например: *йоьлла* → *йо*. После выделения основы снова выполняется поиск.

---

### ЭТАП 9. Клик по слову

**Frontend отправляет:**

- `word`
- `tokenId`
- `textId`

**Backend возвращает:**

- перевод
- грамматику
- форму слова

---

### ЭТАП 10. Прогресс слова

Система отслеживает:

- сколько раз пользователь нажимал слово
- выучено ли слово

---

### ЭТАП 11. Прогресс текста

Отслеживается:

- сколько слов изучено
- процент понимания текста

---

### ЭТАП 12. Админский словарь

Администратор может добавлять слова, редактировать, добавлять формы. Это основной источник переводов.

---

### ЭТАП 13. Кэширование

Используется **Redis / DB cache**. Кэшируются:

- переводы
- результаты морфологии

---

### ЭТАП 14. Оптимизация разметки

Для ускорения системы:

- разметка делается **один раз**
- страницы **предварительно разбиты**
- токены **индексированы**

Это позволяет отдавать страницу за миллисекунды.

---

### ЭТАП 15. API для Frontend

| Действие               | Метод | Endpoint                    |
|------------------------|-------|-----------------------------|
| Получить страницу текста | GET   | `/texts/:id/pages/:page`    |
| Получить перевод слова  | POST  | `/words/lookup`             |
| Получить прогресс       | GET   | `/progress/text/:id`        |

**Ответ «страница текста»:** `tokens[]`, `contentRich`

**Ответ «перевод слова»:** `translation`, `grammar`, `baseForm`

---

## 5. Почему система будет быстрой

| # | Причина | Результат |
|---|---------|------------|
| 1️⃣ | Загружается только 1 страница | Не вся книга |
| 2️⃣ | Словарь вызывается только при клике | Нет лишних запросов |
| 3️⃣ | Токены сохранены в базе | Нет повторной разметки |
| 4️⃣ | Есть кэш словаря | Повторные слова мгновенно находятся |

---

## 6. Итог

Система строится вокруг **трёх ключевых идей:**

1. **Предварительная разметка текста**
2. **Ленивая загрузка переводов** (по клику)
3. **Постраничная загрузка текста**

Это делает систему:

- масштабируемой для книг
- очень быстрой
- удобной для изучения языка
